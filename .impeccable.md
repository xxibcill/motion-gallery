## Design Context

### Users
Primary users are frontend engineers, with designers as a secondary audience. They come here as practitioners who need a fast, credible code reference library for motion patterns they can understand, evaluate, and reuse. The main job to be done is helping people who are unsure what animation to use quickly find a strong option and lift a reusable implementation into their own frontend work.

### Brand Personality
The brand should feel easy to select, expert, and fun to interact with. The interface should create confidence through clear organization and strong curation, while still feeling lively and rewarding in the way it responds. The tone should read as technically credible, direct, and polished rather than playful for its own sake.

### Aesthetic Direction
The product should sit near the feel of [motion.dev/examples](https://motion.dev/examples) and [FreeFrontend's JavaScript gallery](https://freefrontend.com/javascript-gallery), but with stronger curation and better reuse clarity. Keep the current strengths already visible in the codebase: immersive surfaces, strong motion, and clear visual grouping. Support both light and dark themes. Use motion to make selection and comparison easier, not just more decorative. Avoid cluttered gallery patterns, novelty-first effects, or visuals that make implementation details harder to scan.

### Design Principles
1. Make choosing fast. Navigation, filtering, categorization, and preview hierarchy should help engineers find a suitable pattern quickly.
2. Show expert taste. Motion, typography, spacing, and color should feel intentional and technically credible, not generic or trend-chasing.
3. Keep reuse obvious. Every demo should clearly communicate what it does, when to use it, and how easily it can be adapted into production code.
4. Make interaction rewarding. Interfaces should feel responsive and fun, but the motion should clarify state and structure rather than compete with them.
5. Build for both accessibility and theme flexibility. Ship light and dark experiences with a WCAG AA baseline for contrast, clear focus states, keyboard access, and reduced-motion fallbacks.
